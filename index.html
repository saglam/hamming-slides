An optimal lower bound for the $k$-Hamming distance problem

Mert Sağlam

University of Washington

Hamming distance defined

$k$-Hamming distance is hard to compute:

  1. Comunication complexity. Any $k$-Hamming distance protocol requires $\Omega(k \log k)$ bits
  2. Parity decision trees. Any randomized parity decision tree has size $\exp(\Omega(k \log k))$
  3. Property testing. Any property tester for k-linearity requires $\Omega(k \log k)$ queries.

  • Alice receives $x\in\{0,1\}^n$.
  • Bob receives $y\in\{0,1\}^n$.
  • With the promise: $\Ham(x,y)\le k$.

An $O(k\log k)$ bits protocol

  • Consider the graph on $V=\{0,1\}^n$
  • There is an edge between $u,v$ if $\Ham(u,v)\le 2k$
  • Alice and Bob fix a proper coloring of this graph
  • Alice sends the color of $x$ over to Bob.
  • Bob declares closest string to $y$ with the given
    color as $x$. This recovers $x$. Suppose not
  • Takes $O(k\log(n/k))$ bits of communication.

An $O(k\log k)$ bits protocol

  • Permute coordinates randomly
  • Replace every next $\delta n/k^2$ bits with their XOR
  • Run the previous protocol on new strings of length $k^2/\delta$
  • Incurs an error probability of $\le \delta$
  • Complexity $O(k\log (k/\delta))$
